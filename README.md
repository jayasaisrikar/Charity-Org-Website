# Charity-Org-Website

## Supabase setup

Create a `contact_submissions` table for the contact form and `/admin` page.
Run [supabase/schema.sql](/Users/jayasaisrikar/Desktop/charity-org/supabase/schema.sql) in the Supabase SQL editor:

```sql
create table contact_submissions (
  id bigint generated by default as identity primary key,
  name text not null,
  email text not null,
  subject text,
  message text not null,
  created_at timestamptz not null default now()
);
```

The app reads `NEXT_PUBLIC_SUPABASE_URL` and `NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY`.
Copy `.env.example` to `.env.local` and set your Supabase project URL.

## Admin login

The `/admin` route is protected by a simple admin login.

- Email: `admin@snehitham.org`
- Password: `Admin#1234`

You can override these with `ADMIN_EMAIL`, `ADMIN_PASSWORD`, and
`ADMIN_SESSION_SECRET` in `.env.local` or your deployment environment.

If row level security is enabled, add policies that match how public you want
this route to be. For a public submissions page:

```sql
alter table contact_submissions enable row level security;

create policy "Allow public contact submissions"
on contact_submissions for insert
to anon
with check (true);

create policy "Allow public submission reading"
on contact_submissions for select
to anon
using (true);
```
